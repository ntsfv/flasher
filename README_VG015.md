## UART загрузчик для микроконтроллера К1921ВГ015

Описание протокола находится в ```NOTES.md```.

### Описание

* Загрузчик работает через UART0 при параметрах: 8 бит, без бита четности
* Скорость UART определяется автоматически при инициализации соединения
* Системная частота 50 МГЦ (выход PLL, работающей от внутреннего источника 16 МГц)
* Загрузчик позволяет принимать непрерывный поток данных
* Реализованы команды:
    - запрос информации о микроконтроллере и загрузчике
    - запрос CFGWORD
    - модификация CFGWORD (при сохранении остальной информации на странице)
    - чтение страницы (как основной области, так и NVR)
    - запись страницы (как основной области, так и NVR)
    - стирание страницы (как основной области, так и NVR)
    - стирание всей основной области
* Опционально, запись страницы можно проводить с её предварительным стиранием
* Команда модификации CFGWORD производит контроль записанной информации
* Загрузчик не позволяет осуществлять чтение, стирание или запись страниц, где он расположен

Когда загрузчик зашит и активирован - микроконтроллер по включению питания всегда проходит через него. Если на момент включения был активен BOOTEN, то осуществится переход к исполнению тела загрузчика и ожидание связи по UART. Если BOOTEN был неактивен, то с помощью записи в регистр контролера флэша BDIS и программного сброса ядра произойдёт переход к исполнению пользовательской программы c 0x80002000.
При линковке следует учитывать исполнение пользовательской программы с адреса 0x80002000.

# Сборка и загрузка

Требуется:

* make
* openocd

```bash
# -- Сборка
cd firmware/k1921vg015
./build.sh

# -- Загрузка в микроконтроллер
./flash.sh
```

### Подключение

|  К1921ВГ015   | USB-UART |
|:-------------:|:--------:|
| UART0 TX      | RX       |
| UART0 RX      | TX       |
| BOOTEN        | RTS      |
| RST           | DTR      |

```RTS``` и ```DTR``` подключать не обязательно, но рекомендуется, т.к. это позволит автоматически управлять сбросом микроконтроллера и входом в режим загрузчика.
В противном случае последовательность действий следующая:

* Подключаем ```RX``` и ```TX```
* Подтягиваем ```BOOTEN``` к земле
* Сбрасываем микроконтроллер
* Подключаемся через ПО
* Ждём пока появится сообщение о том, что устройство подключено
* Снимаем подтяжку с ```BOOTEN```
* Появится сообщение о том, что устройство готово
* Выполняем дальнейшие операции
