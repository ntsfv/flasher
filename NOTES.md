
# Описание протокола

## Общие правила обмена

* Программа на ПК - (_**H**_)ost, загрузчик в микроконтроллере - (_**D**_)evice
* Только _**H**_ инициирует обмен, _**D**_ ему отвечает
* Обмен производится командами
* Команда - это требуемое к исполнению действие или сообщение, которое умеет уникальный код
* _**D**_ отправляет команду-статус завершения последней операции
* Каждая команда запакована в пакет
* В каждом пакете находится только одна команда
* Передача пакета осуществляется от младшего байта к старшему (little endian)
* Каждый пакет начинается с 16-битной сигнатуры начала пакета
* Сигнатуры разные для направлений от _**H**_->_**D**_ и _**D**_->_**H**_
* Остальной формат пакета одинаков для обоих направлений
* Каждый пакет защищён CRC16 в конце пакета
* CRC16 считается по всему телу пакета кроме сигнатуры
* _**D**_ способен принимать непрерывный поток пакетов без пауз за счёт внутреннего FIFO
* При инициализации обмена _**H**_ отсылает специальный код 0x7F, по которому _**D**_ конфигурирует скорость UART
* После инициализации UART _**D**_ пересылает свою сигнатуру с обменянными местами байтами (чтобы она не была принята за начало пакета)
* После подтверждения соединения выполняется инициализация самого загрузчика _**D**_, по окончанию которой будет отправлено сообщение о готовности

## Структура пакета

|Адрес |Размер  |Описание                                                                                                                          |
|------|--------|--------------------------------------------------------------------------------------------------------------------------------- |
|0     |2       |Сигнатура начала команды, для передачи в устройство 0x5C81 (_**H**_->_**D**_), для приёма из устройства 0x7EA3 (_**D**_->_**H**_) |
|2     |1       |Код команды, который задаёт тип действия или события                                                                              |
|3     |1       |Битовая инверсия кода команды для проверки                                                                                        |
|4     |2       |Длина N блока дополнительной информации в байтах (N кратно 4)                                                                     |
|6     |N       |Блок дополнительной информации (содержимое зависит от кода команды)                                                               |
|6+N   |2       |Рачет CRC16 пакета с адреса 2 по 6+(N-1) включительно (исключая сигнатуру)                                                        |

Таким образом каждый пакет состоит из 8 служебных байт и N байт информации.

```
Порядок передачи пакета:

   lsb     msb                            lsb     msb     lsb                 msb     lsb     msb
 _______ ________ _________ __________ _______ ________ _______ ___ ... ___ _______ _______ ________
|       |        |         |          |       |        |       |           |       |       |        |
|   сигнатура    |   Код   | Инв. код |     Длина      |     Блок дополнительной   |     CRC16      |
|                | команды | команды  |       N        |         информации        |                |
|_______|________|_________|__________|_______|________|_______|___ ... ___|_______|_______|________|
    0        1        2         3         4        5       6                6+(N-1)   6+N    6+(N+1)
```

## Команды

### [0x35] CMD_GET_INFO - Запрос информации
**Направление _H_->_D_**
Запрос информации от _**D**_.

* N: 0
* Доп. информация: -

**Направление _D_->_H_**
Команда-сообщение ```CMD_MSG``` со следующими параметрами

* N = 16
* Доп. информация:
    - 1 байт - сообщение ```MSG_OK```
    - 1 байт - код исходной команды (```CMD_GET_INFO```)
    - 2 байта - с кодом пустых данных (```0x55```)
    - 4 байта - ```SIU->CHIPID``` (для К1921ВК01Т - ```0x00000000```)
    - 4 байта - ```SCB->CPUID```
    - 4 байта - версия загрузчика

Если зафиксированно различие контрольных сумм, то

* N: 4
* Доп. информация:
    - 1 байт - сообщение ```MSG_ERR_CRC```
    - 1 байт - код исходной команды (```CMD_GET_INFO```)
    - 2 байта - с кодом пустых данных (```0x55```)

### [0x65] CMD_SET_CFGWORD - Установка конфигурационного слова
**Направление _H_->_D_**
Установка конфигурационного слова CFGWORD в _**D**_. Страница на которой оно находится копируется в ОЗУ и производится модификация. Затем страница стирается и пишется данными из ОЗУ (адреса с пустыми данными 0xFFFFFFFF пропускаются). Затем вся страница читается снова и происходит верификация записанного, с уже имеющимся дампом. Если модификация области запрещена через CFGWORD, то будет возвращен код ```MSG_FAIL```, а чтение, модификация, стирание и запись производиться не будут.

**К1921ВК035**

* N: 4
* Доп. информация:
    - 4 байта - CFGWORD

**К1921ВК01Т**

* N: 52
* Доп. информация:
    - 4 байта - CFGWORD
    - 16 байт - BFLOCK
    - 32 байта - UFLOCK

**К1921ВК028**

* N: 8
* Доп. информация:
    - 8 байт - CFGWORD

**Направление _D_->_H_**
Команда-сообщение ```CMD_MSG``` со следующими параметрами

**К1921ВК035**

* N: 8
* Доп. информация:
    - 1 байт - сообщение ```MSG_OK``` если операция успешна, и ```MSG_FAIL``` в противном случае; сообщение ```MSG_ERR_CRC``` если зафиксированно различие контрольных сумм
    - 1 байт - код исходной команды (```CMD_SET_CFGWORD```)
    - 2 байта - с кодом пустых данных (```0x55```)
    - 4 байта - CFGWORD из исходной команды

**К1921ВК01Т**

* N: 56
* Доп. информация:
    - 1 байт - сообщение ```MSG_OK``` если операция успешна, и ```MSG_FAIL``` в противном случае; сообщение ```MSG_ERR_CRC``` если зафиксированно различие контрольных сумм
    - 1 байт - код исходной команды (```CMD_SET_CFGWORD```)
    - 2 байта - с кодом пустых данных (```0x55```)
    - 4 байта - CFGWORD из исходной команды

### [0xA5] CMD_READ_PAGE - Чтение страницы flash
**Направление _H_->_D_**
Чтение страницы данных из _**D**_. Адрес должен указывать на начало страницы. Невыравненный адрес автоматически будет выравнен на стороне _**D**_. Если страница находится в основном массиве flash, то 7 бит конфигурации должен быть сброшен. Если необходимо чтение из NVR области - 7 бит должен быть установлен. Состояние 5-го бита конфигурации определяет flash, к которой будет произведен доступ. Если чтение запрещено с помощью CFGWORD, то будет возвращен код ```MSG_FAIL```.

* N: 4
* Доп. информация:
    - 3 байта - адрес, по которому определяется номер страницы
    - 1 байт - конфигурация: 7 бит определяет область работы [0-основная, 1-NVR], 5 бит определяет flash [0-BOOTFLASH, MFLASH, 1-USERFLASH, BFLASH]

**Направление _D_->_H_**
Команда-сообщение ```CMD_MSG``` со следующими параметрами

* N: 8 + ```PAGE_BYTES```
* Доп. информация:
    - 1 байт - сообщение ```MSG_OK``` если операция успешна, и ```MSG_FAIL``` в противном случае; сообщение ```MSG_ERR_CRC``` если зафиксированно различие контрольных сумм
    - 1 байт - код исходной команды (```CMD_READ_PAGE```)
    - 2 байта - с кодом пустых данных (```0x55```)
    - 3 байта - адрес из исходной команды
    - 1 байт - конфигурация из исходной команды
    - ```PAGE_BYTES``` байт - одна страница данных, размером ```PAGE_BYTES```


### [0x9A] CMD_WRITE_PAGE - Запись страницы flash
**Направление _H_->_D_**
Страничная запись данных в _**D**_. Адрес должен указывать на начало страницы. Невыравненный адрес автоматически будет выравнен на стороне _**D**_. Состояние 5-го бита конфигурации определяет flash, к которой будет произведен доступ. Перед записью страница стирается, если установлен 6 бит конфигурационного байта. Если страница находится в основном массиве flash, то 7 бит конфигурации должен быть сброшен. Если необходима запись в NVR область - 7 бит должен быть установлен. Если номер страницы соответсвует расположению загрузчика или соответсвующая область защищена через CFGWORD, то страница стерта и записана не будет, и будет возвращен код ```MSG_FAIL```.

* N: 4 + ```PAGE_BYTES```
* Доп. информация:
    - 3 байта - адрес, по которому определяется номер страницы
    - 1 байт - конфигурация: 7 бит определяет область работы [0-основная, 1-NVR], 6 бит определяет необходимость стирания страницы перед записью [0-не стирать, 1-стирать], 5 бит определяет flash [0-BOOTFLASH, MFLASH, 1-USERFLASH, BFLASH]
    - ```PAGE_BYTES``` байт - одна страница данных, размером ```PAGE_BYTES```

**Направление _D_->_H_**
Команда-сообщение ```CMD_MSG``` со следующими параметрами

* N: 8
* Доп. информация:
    - 1 байт - сообщение ```MSG_OK``` если операция успешна, и MSG_FAIL в противном случае; сообщение ```MSG_ERR_CRC``` если зафиксированно различие контрольных сумм
    - 1 байт - код исходной команды (```CMD_WRITE_PAGE```)
    - 2 байта - с кодом пустых данных (```0x55```)
    - 3 байта - адрес из исходной команды
    - 1 байт - конфигурация из исходной команды

### [0xFA] CMD_MSG - Сообщение статуса
**Коды сообщений**

* ```[0x01] MSG_ERR_CMD``` - неправильно принятый или неизвестный код команды
* ```[0x02] MSG_ERR_CRC``` - ошибка сравнения контрольных сумм
* ```[0x03] MSG_READY``` - загрузчика готов к обмену
* ```[0x04] MSG_OK``` - операция выполнена успешно
* ```[0x05] MSG_FAIL``` - операция не выполнена

**Направление _H_->_D_**
Отсутствует.

**Направление _D_->_H_**
Статус или сообщение о событии, отправляемое в **_H_**. В общем виде выглядит как

* N: зависит от исходной команды
* Доп. информация:
    - 1 байт - код сообщения
    - 1 байт - код исходной команды
    - X байт - содержимое зависит от исходной команды

Если принят неверный или неизвестный код команды, то

* N: 4
* Доп. информация:
    - 1 байт - ```MSG_ERR_CMD```
    - 1 байт - ```[0x00]CMD_NONE```
    - 2 байта - с кодом пустых данных (```0x55```)

Если инициализация загрузчика прошла успешно, то

* N: 4
* Доп. информация:
    - 1 байт - ```MSG_READY```
    - 1 байт - ```[0x00]CMD_NONE```
    - 2 байта - с кодом пустых данных (```0x55```)
