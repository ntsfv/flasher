## UART загрузчик для серии микроконтроллеров К1921ВГXT (К1921ВГ1T, К1921ВГ3T, К1921ВГ5T, К1921ВГ7T)

Описание протокола находится в ```NOTES.md```.

### Описание

* Загрузчик работает через UART0 при параметрах: 8 бит, без бита четности
* Скорость UART определяется автоматически при инициализации соединения
* Системная частота для каждого контроллера определяется настройками в makefile
    - -D SYSCLK_PLL или -D SYSCLK_HSE - определяет источник тактирования PLL или HSE, соответсвенно (по умолчанию PLL)
    - -D HSECLK_VAL=16000000 - определяет величину опорного кварцевого резонатора HSE.
    - Результрующая частота для каждого контроллера определяется максимально допустимой. Контроллер (при использовании PLL) старается работать на 
      максимально разрешенной частоте.
* Загрузчик позволяет принимать непрерывный поток данных
* Реализованы команды:
    - запрос информации о микроконтроллере и загрузчике
    - запрос CFGWORD
    - модификация CFGWORD (при сохранении остальной информации на странице)
    - чтение страницы (как основной области, так и NVR)
    - запись страницы (как основной области, так и NVR)
    - стирание страницы (как основной области, так и NVR)
    - стирание всей основной области
* Опционально, запись страницы можно проводить с её предварительным стиранием
* Команда модификации CFGWORD производит контроль записанной информации
* Загрузчик не позволяет осуществлять чтение, стирание или запись страниц, где он расположен

Когда загрузчик зашит и активирован - микроконтроллер по включению питания всегда проходит через него. Если на момент включения был активен BOOTEN, то осуществится переход к исполнению тела загрузчика и ожидание связи по UART. Если BOOTEN был неактивен, то с помощью записи в регистр контролера флэша BDIS и программного сброса ядра произойдёт переход к исполнению пользовательской программы c 0x00002000.
При линковке следует учитывать исполнение пользовательской программы с адреса 0x00002000.

Следует обратить внимание, что расположение загрузочного UART и пина BOOTEN определяется в файле boot_conf.h для каждого микроконтроллера серии отдельно. При желании, возможно изменить расположение выводов.

По умолчанию:

К1921ВГ1Т:
    BOOTEN: A8
    UART:
        TX: B1
        RX: B0
К1921ВГ3Т:
    BOOTEN: A8
    UART:
        TX: C1
        RX: C0
К1921ВГ5Т:
    BOOTEN: A8
    UART:
        TX: B1
        RX: B0
К1921ВГ7Т:
    BOOTEN: B0
    UART:
        TX: A1
        RX: A0

# Сборка и загрузка

Требуется:

* make
* openocd

* Выбор микроконтроллера, для которого будет осуществлятся сборка осуществляется настройкой значения TARGET_IDX в makefile.

```bash
# -- Сборка
cd firmware/k1921vgxt/source/bin
make all

# -- Загрузка в микроконтроллер
# -- Вместе со сборкой
cd firmware/k1921vgxt/source/bin
# -- Сервер openocd должен быть активен
make load 

```

* Готовые .elf файлы сборки можно найти в firmware/k1921vgxt/releases

### Подключение

|  К1921ВГXT    | USB-UART |
|:-------------:|:--------:|
| UART  TX      | RX       |
| UART  RX      | TX       |
| BOOTEN        | RTS      |

* Подключаем ```RX``` и ```TX```
* Подтягиваем ```BOOTEN``` к земле
* Сбрасываем микроконтроллер
* Подключаемся через ПО
* Ждём пока появится сообщение о том, что устройство подключено
* Снимаем подтяжку с ```BOOTEN```
* Появится сообщение о том, что устройство готово
* Выполняем дальнейшие операции
